# 写脚本

我之前也写过一点点的bash shell，完全是为了少打一些命令。但是我之前
写的代码很不规范，今天看了第一份代码，立刻发现了我应该注意的地方。

- 注释\#后面要加一个空格，一行写不完的注释后面几行的要以\#+开头。

``` {.example}
# this is a comment,but it may be
#+ too long
```

- 程序的结束要写一个exit 0，这个就像C语言中的return 0
- 引用变量，最好要加上双引号
- 引用命令的执行结果使用\`\`（和波浪号同一个键的那个符号）
- \#!这个叫sha-bang
